# BTC Multi-Timeframe RSI Dashboard

Аналитический Streamlit-дашборд: оценивает **степень готовности** рынка BTC к развороту,
сопоставляя RSI(14) на четырех таймфреймах. Без автоторговли, без API-ключей —
только публичные котировки.

## Идея

RSI на одном таймфрейме часто дает ложные сигналы: на 15m он показывает «перепроданность»,
пока на 1h/4h/1d падение только разгоняется. Здесь каждый таймфрейм играет роль:

| ТФ  | Роль                     | Вес  |
|-----|--------------------------|------|
| 15m | Ранний триггер           | 0.15 |
| 1h  | Подтверждение импульса   | 0.20 |
| 4h  | Основной контекст рынка  | 0.35 |
| 1d  | Глобальный фильтр        | 0.30 |

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется http://localhost:8501. Источник данных выбирается в сайдбаре:
Binance через ccxt (по умолчанию), прямой Binance REST, Bybit/OKX (если Binance
недоступен в регионе) или Demo (синтетика, офлайн).

## Метрики

- **Long / Short Readiness (0–100)** — взвешенная по таймфреймам готовность к лонгу/шорту.
- **MTF Alignment (0–100)** — насколько таймфреймы согласованы по направлению.
- **Signal Confidence (0–100)** — 0.65 × преобладающая readiness + 0.35 × alignment,
  со штрафом-«вето» от дневного таймфрейма.

## Светофор

- 🟢 green — сигнал подтвержден старшими ТФ (4h поддерживает, дневка не против);
- 🟡 yellow — младшие ТФ сработали, старшие не подтвердили: ждать;
- 🔴 red — старшие ТФ против или согласованного сигнала нет.

## Дисклеймер

Инструмент показывает готовность рынка, а не точки входа. Не финансовый совет.
