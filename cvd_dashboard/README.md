# BTC CVD Dashboard — Cumulative Volume Delta

Аналитический Streamlit-дашборд: показывает **поток агрессивных ордеров** по BTC
(Binance USDT-M Futures) — кто реально двигает цену маркет-ордерами.
Без автоторговли, без API-ключей — только публичные эндпоинты.

Скрипт полностью автономен: вся логика в одном файле `app.py`,
никаких зависимостей от других модулей проекта.

## Идея

Из klines Binance берется поле *taker buy volume* — объем, купленный агрессорами:

```
Delta = маркет-покупки − маркет-продажи = 2 × taker_buy − volume
CVD   = накопленная сумма дельты
```

Главный сигнал — **дивергенция цены и CVD**:

| Цена            | CVD             | Сигнал                                         |
|-----------------|-----------------|------------------------------------------------|
| новый минимум   | минимум НЕ обновлен | 🐂 бычья дивергенция — продавцы выдыхаются |
| новый максимум  | максимум НЕ обновлен | 🐻 медвежья — рост без поддержки покупок  |
| новый максимум  | новый максимум  | ✅ рост подтвержден покупками                  |
| новый минимум   | новый минимум   | 🔻 падение подтверждено продажами              |

Анализ на четырех таймфреймах с ролями, как в `btc_rsi_dashboard`:
15m — ранний сигнал, 1h — подтверждение, 4h — основной контекст, 1d — глобальный поток.

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется http://localhost:8501. Если Binance недоступен (сеть, гео-блокировка) —
дашборд переключается на синтетические demo-данные и явно это помечает.

## Светофор

- 🟢 green — поток подтверждает движение цены, дивергенций нет;
- 🟡 yellow — дивергенция на младших ТФ (15m/1h): ранний сигнал, ждать 4h/1d;
- 🔴 red — дивергенция на старших ТФ (4h/1d): поток против цены, вероятен разворот.

## Дисклеймер

CVD показывает поток ордеров, а не точки входа. Не финансовый совет.
